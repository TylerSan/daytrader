# Project Rules — DayTrader Platform

## Dependency Policy

- **Prefer proven third-party tools, libraries, and skills** over building from scratch. Don't reinvent the wheel.
- Before adopting any third-party dependency:
  1. Verify it is actively maintained (recent commits, responsive maintainers)
  2. Check for known security vulnerabilities (CVEs, advisory databases)
  3. Confirm license compatibility (MIT, Apache 2.0, BSD preferred)
  4. Prefer libraries with strong community adoption (stars, downloads, usage)
- Only build custom when no suitable existing solution exists or existing options have critical flaws.
- Document the rationale when choosing to build vs. reuse.
